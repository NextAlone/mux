---
title: .muxignore
description: Sync ignored files to JJ Workspace checkouts
---

JJ workspaces share repository history but **not** untracked or ignored files. This means files like `.env` that are in your `.gitignore` won't be available in isolated workspace checkouts.

A `.muxignore` file lets you declare which ignored files should be copied into new JJ Workspace checkouts.

## Setup

Create a `.muxignore` file in your project root (alongside `.gitignore`):

```gitignore
# Sync these ignored files to JJ Workspace checkouts
!.env
!.env.local
!config/secrets.json
```

Only lines prefixed with `!` (negation) are actionable — they identify ignored files to copy. Comments (`#`) and blank lines are ignored.

> Mux only copies files that are already ignored in your project and currently exist on disk.

## Behavior

- **When:** Files are copied after the jj workspace is created, before [init hooks](/hooks/init) run — so your init scripts have access to `.env` and other secrets.
- **Source:** Your project root directory (the main checkout).
- **Copy mode:** Physical file copy, not symlink. Each workspace gets its own independent copy.
- **No overwrite:** Files that already exist in the workspace are not overwritten.
- **Best-effort:** If a file can't be copied, Mux logs debug information and continues — it won't block workspace creation.

## Where to place it

Place `.muxignore` in the project root, next to your `.gitignore`. You may want to commit it to your repo so all collaborators share the same sync rules.
