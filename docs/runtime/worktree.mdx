title: Worktree Runtime
description: Isolated jj workspace environments for parallel agent work

Worktree runtime uses [jj workspaces](https://jj-vcs.github.io/jj/latest/working-copy/#workspaces) on your local machine.

Each workspace gets its own directory, while all workspaces share the same colocated jj repository.

## How worktrees work

A workspace is a separate directory on the same filesystem as the main repository.

- It has its own working copy commit and files on disk.
- It shares repository history with the main checkout through jj.

Because the jj repository is shared, **changes created in any workspace are immediately visible to your other workspaces (including your main checkout).**

Workspaces aren’t locked to a bookmark. The agent can create new changes and move bookmarks as needed. If you want to constrain bookmark behavior, do it via your [instruction files](/agents/instruction-files).

## Filesystem layout

Worktree runtime workspaces are stored in `~/.mux/src/<project-name>/<workspace-name>`.

Example layout:

```text
~/.mux/src/
  mux-main/
    improved-auth-ux/
    fix-ci-flakes/
```
